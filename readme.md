# 基于 STM32 的智能门禁系统设计

本项目为本科毕业设计项目，主要实现基于 STM32 的智能门禁系统。系统以 STM32F103ZET6 作为主控芯片，结合 K210 视觉识别模块、GY-906 红外测温模块、OLED 显示模块、蜂鸣器、语音模块和舵机等外设，完成基本的人脸识别、体温判断、状态显示、异常提示和模拟开门控制功能。

## 项目简介

传统门禁系统通常采用机械钥匙、刷卡或密码输入等方式完成开门控制，在身份确认和异常状态提示方面存在一定局限。本项目在普通门禁控制基础上加入人脸识别和红外测温功能。

系统运行时，K210 负责采集人脸图像并进行识别，然后通过 UART 串口将识别结果发送给 STM32。STM32 接收到识别结果后，继续读取 GY-906 红外测温模块的数据，并根据人脸识别结果和体温结果判断是否控制舵机模拟开门。

当人脸识别成功且体温处于设定范围内时，STM32 输出 PWM 信号控制舵机转动，完成模拟开门；当人脸识别失败或体温异常时，系统不执行开门动作，并通过 OLED、蜂鸣器和语音模块进行提示。

温湿度、烟雾和光照检测作为辅助环境监测功能，主要用于本地数据显示和异常提醒，不直接参与门禁开门判断。

## 系统功能

- K210 人脸识别
- STM32 主控逻辑判断
- GY-906 红外测温
- OLED 本地状态显示
- 蜂鸣器异常报警
- SU-03T 语音提示
- 舵机模拟开门
- DHT11 温湿度检测
- MQ-2 烟雾检测
- 光敏电阻光照检测
- STM32 与 K210 串口通信
- 人脸录入和删除指令交互

## 系统整体流程

1. 系统上电后，STM32 初始化各功能模块。
2. K210 采集摄像头图像并进行人脸识别。
3. 若人脸识别失败，系统不开门，并通过 OLED 和语音模块提示。
4. 若人脸识别成功，K210 通过 UART 将识别结果发送给 STM32。
5. STM32 读取 GY-906 红外测温数据。
6. 若体温正常，STM32 控制舵机模拟开门。
7. 若体温异常，系统不开门，并触发 OLED 显示、蜂鸣器报警和语音提示。
8. 温湿度、烟雾和光照模块持续采集环境数据，并在 OLED 上显示。
9. 当烟雾浓度超过设定阈值时，系统通过 OLED 和蜂鸣器进行异常提示。

## 硬件组成

| 模块 | 作用 |
|---|---|
| STM32F103ZET6 | 主控芯片，负责逻辑判断、传感器读取和外设控制 |
| K210 视觉模块 | 负责人脸检测、特征提取和识别结果输出 |
| GY-906 红外测温模块 | 采集通行人员体温数据 |
| OLED 显示模块 | 显示识别状态、体温、环境数据和异常信息 |
| SU-03T 语音模块 | 播放通行、识别失败、体温异常等语音提示 |
| 蜂鸣器 | 用于异常报警 |
| 舵机 | 模拟门禁开门动作 |
| DHT11 | 采集环境温湿度数据 |
| MQ-2 | 检测烟雾或可燃气体异常 |
| 光敏电阻 | 检测环境光照强度 |

## 软件环境

### STM32 端

- Keil MDK-ARM
- STM32CubeMX
- STM32 HAL 库
- ST-Link 下载与调试

STM32 程序主要完成 GPIO、ADC、IIC、UART、定时器和 PWM 等外设初始化，并负责任务调度、串口通信、传感器数据读取、OLED 显示、蜂鸣器报警、语音提示和舵机控制。

### K210 端

- MaixPy IDE
- Python
- KPU 硬件加速单元

K210 程序主要完成人脸图像采集、人脸检测、特征提取、人脸比对和识别结果发送。

## 通信协议说明

STM32 与 K210 之间采用 UART 串口通信。

### K210 发送给 STM32 的数据帧

| 功能 | 数据帧 | 说明 |
|---|---|---|
| 人脸录入成功 | `AF 05 00 FA` | K210 完成人脸录入后发送 |
| 人脸识别成功 | `AF 02 00 FA` | K210 识别成功后发送 |
| 人脸删除成功 | `AF 01 00 FA` | K210 完成人脸删除后发送 |

### STM32 发送给 K210 的数据帧

| 功能 | 数据帧 | 说明 |
|---|---|---|
| 删除人脸 | `A1 01 02 1A` | STM32 通知 K210 删除人脸 |
| 录入人脸 | `A1 02 00 1A` | STM32 通知 K210 录入人脸 |

STM32 接收数据后会先判断帧头和帧尾是否正确。如果数据帧格式错误，则丢弃该帧数据，避免错误数据直接触发开门动作。
